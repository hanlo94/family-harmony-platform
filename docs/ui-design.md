## 1. 设计理念

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **一眼看懂** | 打开 App 就知道今天要做什么，不需要教程 |
| **温暖不幼稚** | 大人和孩子都觉得舒服，不偏向任何年龄段 |
| **动效有意义** | 完成任务的动画让人有成就感，不是为了炫技 |
| **减少摩擦** | 3 步内完成核心操作，表单字段越少越好 |

### 1.2 情绪板关键词

`温暖` `轻量` `和谐` `可靠` `小确幸`

| `--color-surface` | `#F5EDE4` | 卡片、列表项背景 |

| `--color-white` | `#FFFFFF` | 纯白卡片、输入框背景 |

### 2.2 字体

移动端 H5 + 小程序，优先使用系统字体以保证加载速度和原生体验：

```css
/* 中文正文字体栈 */
--font-body: -apple-system, "PingFang SC", "Hiragino Sans GB",
             "Microsoft YaHei", "Helvetica Neue", sans-serif;

/* 数字/英文显示字体（用于时间、计数、难度数字） */
--font-display: "SF Pro Rounded", "DIN Alternate", -apple-system,
                "PingFang SC", sans-serif;
```

**字体层级**：

| 级别 | 大小 | 字重 | 用途 |
|------|------|------|------|
| H1 | 22px / Bold (700) | 页面标题（如"家庭任务"） |
| H2 | 18px / Semibold (600) | 区块标题（如"待完成"） |
| Body | 16px / Regular (400) | 正文、任务标题 |
| Caption | 13px / Regular (400) | 辅助信息、时间、成员名 |
| Small | 11px / Medium (500) | 标签、徽章 |

### 2.3 间距

基于 4px 栅格，移动端适配：

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-xs` | 4px | 图标与文字间距 |
| `--space-sm` | 8px | 标签内边距 |
| `--space-md` | 12px | 列表项内边距 |
| `--space-lg` | 16px | 卡片内边距、页面水平边距 |
| `--space-xl` | 20px | 区块间距 |
| `--space-2xl` | 24px | 页面顶部/底部留白 |

### 2.4 圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-sm` | 6px | 标签、徽章、小按钮 |
| `--radius-md` | 10px | 卡片、列表项、输入框 |
| `--radius-lg` | 14px | 大按钮（主 CTA） |
| `--radius-full` | 9999px | 头像、药丸形标签 |

### 2.5 阴影

移动端阴影克制使用，主要通过颜色区分层级：

| Token | 值 | 用途 |
|-------|-----|------|
| `--shadow-card` | `0 1px 3px rgba(0,0,0,0.06)` | 卡片浮起 |
| `--shadow-fab` | `0 2px 8px rgba(0,0,0,0.12)` | 浮动操作按钮 |

## 3. 页面清单与路由

```
┌──────────────────────────────────────────────────────────┐
│                      App Shell                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  📋 任务  │  │  👨‍👩‍👧 家庭 │  │  👤 我的  │   ← Bottom   │
│  │  (首页)   │  │          │  │          │     TabBar   │
│  └──────────┘  └──────────┘  └──────────┘              │
└──────────────────────────────────────────────────────────┘
```

| 页面 | 路由 | 说明 |
|------|------|------|
| 任务首页 | `/pages/index/index` | TabBar 首页，展示当前家庭任务列表，含逾期/临近到期专区 |
| 任务详情 | `/pages/task/detail?id=xxx` | 查看任务完整信息、操作按钮、驳回原因 |
| 创建任务 | `/pages/task/create` | 模板选择 → 表单填写（含重复规则、验收开关） |
| 验收任务 | `/pages/task/verify?id=xxx` | 组织者验收（看照片 + 通过/驳回） |
| 家庭管理 | `/pages/family/index` | TabBar 第二个 Tab，成员列表、轻量统计、邀请入口 |
| 邀请成员 | `/pages/family/invite` | 生成邀请链接/二维码 |
| 加入家庭 | `/pages/family/join?code=xxx` | 被邀请者入口 |
| 个人中心 | `/pages/profile/index` | TabBar 第三个 Tab，我的资料、切换家庭、设置入口 |
| 提醒设置 | `/pages/profile/settings` | 提醒开关 Toggle 🆕 |
| 登录页 | `/pages/login/index` | 微信登录入口 |

**交互说明**：
- 逾期专区：当存在逾期任务时，在筛选 Tab 上方以 Rose 色横条突出展示
- 临近到期专区：在逾期专区之后，以 Honey 色背景展示截止前 1 小时内的任务
- 逾期任务卡片：左侧红色竖线标记，截止时间显示为红色，右下角显示"已逾期 Xh"
- 临近到期任务卡片：左侧黄色竖线标记，截止时间显示"还剩 X 分钟"
- 普通待完成任务卡片：显示重复规则 Badge（"每天"/"每周"）和验收标记（"🔍验收"）
- 顶部 Tab 切换：包含待完成、待验收、已驳回、已完成、已取消
- 下拉刷新：逾期和临近到期区域同步刷新
- 空状态：显示插画 + "今天没有待完成的任务，太棒了！"
- FAB 按钮：CHILD 角色不可见，MEMBER/ORGANIZER 可见

## 7. 动效设计

| 场景 | 动效 | 时机 | 时长 |
|------|------|------|------|
| **任务完成** | 卡片向右滑出 + 绿色对勾缩放出现 | 点击"标记完成"后 | 400ms |
| **验收通过** | 卡片背景变 Moss 绿 + 轻微弹跳 | 点击"验收通过"后 | 500ms |
| **驳回** | 卡片微颤（shake）+ 驳回原因淡入 | 驳回操作 | 300ms |
| **创建任务** | FAB 按钮缩放消失 → 表单从底部滑入 | 点击 FAB | 250ms |
| **Tab 切换** | 内容左右滑动（translateX）+ 底横线跟随 | 切换筛选 Tab | 200ms |
| **列表加载** | 卡片从下往上依次淡入（stagger） | 首次加载 | 每张 80ms 间隔 |
| **下拉刷新** | 顶部加载指示器旋转 + "松手刷新" | 下拉 | — |
| **Toast** | 从顶部滑入 → 2s 后滑出 | 操作反馈 | 300ms |
| **空状态** | 插画轻微浮动（translateY 循环） | 页面空闲 | 3s 循环 |
| **逾期警告** | 逾期横条从顶部展开 + 玫瑰色脉冲 🆕 | 存在逾期任务时 | 400ms |

### 动效原则
- 所有动效尊重 `prefers-reduced-motion`，用户开启减少动效时禁用
- 使用 CSS `transition` / `animation` 而非 JS 动画库，确保 60fps
- 微信小程序版使用 uni-app 的 `<transition>` 组件替代 CSS transition

## 9. 视觉签名（Signature Element）

> **完成时刻的"小确幸"动画**：当成员标记任务完成时，屏幕中央短暂出现一个金色星星（Honey 色）带着粒子效果绽放，持续约 1 秒。这不是游戏化的积分系统，而是一个纯粹的情感反馈——"你为家庭做了贡献，这很重要"。
>
> 这个星星是整个 App 的视觉签名，出现在：
> - 任务完成时（大颗，中央绽放）
> - 验收通过时（小颗，卡片上方）
> - 空状态时（静态，插画中点缀）
