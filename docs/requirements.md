# 产品需求文档 (PRD)

## 家庭共享任务与"无唠叨"协作平台

**版本**：v1.0 MVP  
**日期**：2026-06-10  
**状态**：初稿

---

## 1. 产品愿景

> 让家庭家务分配更公平、协作更愉快，用系统提醒代替人际催促，消除家庭中的"精神负担"。

**一句话**：一个智能分配家务、自动提醒、让贡献看得见的家庭协作 App。

---

## 2. 目标用户画像

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| **家庭组织者** | 通常是家庭中负责安排家务的人（多为妈妈） | 希望公平分配家务，不用反复催促家人，减少精神负担 |
| **家庭成员** | 家庭中其他需要参与家务的人（伴侣、大孩子） | 清楚知道自己要做什么、什么时候做，获得正向反馈 |
| **孩子** | 有一定自理能力的儿童（8岁以上） | 通过完成任务获得成就感，培养责任感 |

---

## 3. MVP 范围定义

### MVP（v1.0）核心流程：任务分配 + 完成追踪

```
家庭创建 → 成员加入 → 创建任务 → 分配任务 → 系统自动提醒 → 成员完成 → 确认验收
```

### 不在 MVP 范围内（后续版本）

| 功能 | 优先级 | 备注 |
|------|--------|------|
| 积分与游戏化激励 | P1 | MVP 后第一优先级 |
| 贡献可视化图表 | P1 | MVP 后第一优先级 |
| 购物清单 | P2 | v2.0 功能 |
| 共享日历 | P2 | v2.0 功能 |
| 智能任务分配算法 | P2 | MVP 用人工分配 |

## 6. 验收标准

### 6.1 家庭创建与成员管理

| AC-ID | 验收标准 |
|-------|----------|
| AC-01 | 用户可通过微信扫码登录，3步内完成首次注册 |
| AC-02 | 家庭组织者可在 1 分钟内创建家庭，自动生成邀请链接 |
| AC-03 | 被邀请者打开链接后可直接加入，无需额外注册流程 |
| AC-04 | 一个用户可以加入多个家庭 |

### 6.2 任务管理

| AC-ID | 验收标准 |
|-------|----------|
| AC-05 | 创建任务表单包含：标题（必填）、描述（选填）、难度 1-5 星、截止时间（必填）、执行人（必填） |
| AC-06 | 任务列表支持按"全部/待完成/已完成"筛选、按执行人筛选 |
| AC-07 | 标记完成时可选择拍照上传，照片压缩后存储 |
| AC-08 | 验收确认后任务状态变为"已完成"，驳回后任务回归"待完成" |

### 6.3 微信通知

| AC-ID | 验收标准 |
|-------|----------|
| AC-09 | 任务到期前 1 小时发送微信模板消息提醒 |
| AC-10 | 用户可在设置中管理是否接收提醒 |
| AC-11 | 通知内容包含：任务名称、截止时间、所属家庭 |

## 7. 非功能性需求

| 类别 | 要求 |
|------|------|
| **性能** | 页面首屏加载 < 2 秒，API 响应 < 500ms（P95） |
| **设备适配** | 移动端优先，适配 iOS/Android 微信内置浏览器 |
| **安全** | 微信 OAuth 2.0 认证，API 鉴权 JWT，HTTPS 传输 |
| **数据** | 用户数据可导出，账号可注销（符合个保法） |
| **可用性** | 核心功能 3 次点击内可达，任务创建流程 < 30 秒 |
| **国际化** | v1.0 仅支持中文（简体） |

---

## 8. MVP 不做（明确边界）

- ❌ 不做积分和兑换系统
- ❌ 不做贡献统计图表
- ❌ 不做购物清单
- ❌ 不做共享日历
- ❌ 不做智能自动分配
- ❌ 不做原生 App（仅微信 H5 / 小程序）
- ❌ 不做多语言

## 9. 术语表
